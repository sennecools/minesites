```typescript
export default async function SubdomainLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ subdomain: string }>;
}) {
  const { subdomain } = await params;
  const server = await db.server.findUnique({
    where: { subdomain },
    select: { theme: true, name: true, serverIp: true },
  });
  const theme = (server?.theme as SiteTheme | null) ?? DEFAULT_THEME;
  const serverName = server?.name ?? subdomain;
  const serverIp = server?.serverIp ?? "";
  // ... render .site-root with theme CSS vars and <SiteNav>
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → DB query | `subdomain` from URL params used in `db.server.findUnique({ where: { subdomain } })` — potential for injection |
| DB theme JSON → CSS inline style | `server.theme` (Json?) cast to SiteTheme and used to build inline CSS vars |
| Server IP → clipboard | serverIp string written to clipboard via navigator.clipboard.writeText |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Injection | layout.tsx db query — subdomain param | accept | Prisma ORM parameterizes all queries; subdomain value is never interpolated into raw SQL. No injection vector. |
| T-02-04 | Tampering | server.theme JSON cast to SiteTheme | mitigate | Cast `server.theme as SiteTheme \| null` then apply `?? DEFAULT_THEME`. If palette key is not in THEME_PRESETS, fall back to DEFAULT_THEME.cyan hex. Validate: `const accent = THEME_PRESETS[theme.palette] ?? THEME_PRESETS.cyan` — prevents unknown key producing `undefined` as a CSS var value. |
| T-02-05 | Injection | CSS var injection — theme values injected inline | accept | CSS custom properties set via React's `style` prop are escaped by React's DOM serializer; they cannot inject `<script>` or break out of the style attribute context. The values are also constrained to hex strings from a static lookup table, not user free-text. |
| T-02-06 | Spoofing | SiteNav serverIp display | accept | serverIp is stored per-server by the authenticated server owner. No spoofing surface for site visitors (they see the stored IP, cannot change it). |
</threat_model>
