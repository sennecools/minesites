This phase delivers UI for **two separate surfaces**. Each has its own design contract.

| Surface | Scope | CSS Boundary |
|---------|-------|-------------|
| Public site (`.site-root`) | Gaming-styled server website visible to visitors | Under `.site-root` class тАФ isolated from dashboard |
| Editor sidebar ("Appearance" tab) | Dashboard controls for server owner to configure theme | Inside existing dashboard styles тАФ no change to `:root` |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact element spacing, swatch gaps |

Exceptions (documented with justification):

| Value | Context | Justification |
|-------|---------|---------------|
| 44px | Minimum tap zone for color swatch circles (`.color-picker`) | WCAG 2.5.5 minimum interactive target size тАФ accessibility exception. The visual circle is 28px (existing pattern); the invisible touch target extends to 44px via padding. |
| 14px | Top and bottom padding inside sticky nav (56px total height) | Optical centering within fixed 56px nav height: 14px top + 28px content + 14px bottom = 56px. The 14px value is driven by the fixed height constraint, not the 8pt grid. |

Source: Standard 8pt scale; existing `.color-picker` (28px) from `globals.css`; WCAG 2.5.5

## Typography

### Public Site (`.site-root`)

**Maximum 2 weights declared.** Body text always uses the legible fallback font regardless of the active gaming font.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 (regular) | 1.6 | system-ui / Inter (legible fallback тАФ never the gaming font) |
| Label / UI copy | 14px | 400 (regular) | 1.4 | system-ui / Inter |
| Heading | 24px | 700 (bold) | 1.2 | `--site-font-display` (active gaming font) |

Weights used: **400** (regular тАФ body and labels) and **700** (bold тАФ headings and display). No other weights.

Primary focal point: server name in hero section at Display size (40px, weight 700, `--site-font-display`).

## Interaction Contracts

### Appearance Tab тАФ Color Swatch Selection

1. User clicks a swatch circle.
2. Selected swatch gets: 2px ring in its own accent color, Framer Motion scale(1.1) snap.
3. Previously selected swatch returns to unselected state instantly.
4. CSS variable `--site-accent` updates immediately on `.site-root` in the preview panel (client-side, no save).
5. Save button activates (if previously disabled). Save persists `server.theme.palette` to DB via `PUT /api/servers/[serverId]`.

### Appearance Tab тАФ Font Selection

1. User clicks a font label.
2. Active label gets: accent-colored bottom border (2px), font-weight 700.
3. `--site-font-display` CSS variable updates immediately in preview panel.
4. Font label renders its own name in the corresponding font (e.g., "Orbitron" label uses the Orbitron font-family).
5. Save button activates. Save persists `server.theme.font` to DB.

### Nav тАФ IP Copy Button

1. User clicks "Copy IP" button.
2. Lucide `Copy` icon тЖТ `Check` icon swap (200ms transition).
3. IP string written to clipboard via `navigator.clipboard.writeText()`.
4. Icon reverts to `Copy` after 2000ms.
5. No toast or modal тАФ icon feedback is sufficient.

Source: CONTEXT.md D-11; existing `Copy`/`Check` import pattern in editor

### Per-Section Background Override

1. Section settings panel shows a "Background" control group at the bottom.
2. Control: color input (HTML `<input type="color">` or hex text field) + "Reset Background" ghost button.
3. When set: section outer wrapper applies `style={{ backgroundColor: settings.backgroundColor }}`.
4. When reset/undefined: section renders `#0e0e10` (site default).
5. Change updates preview immediately (client-side state).

Source: CONTEXT.md D-12

| Interaction | Animation | Duration |

## Constraints

1. `globals.css` `:root` variables must NOT be modified тАФ they are dashboard-only tokens.
2. All public site CSS custom properties are scoped under `.site-root` only.
3. All 5 Google Fonts must be declared statically in `src/app/[subdomain]/layout.tsx` тАФ Next.js `next/font/google` does not support dynamic font declarations at runtime.
4. Theme CSS vars must be injected server-side on the `.site-root` element so public pages have no flash of unstyled content (THEME-04).
5. Dashboard pages never receive the `.site-root` class тАФ isolation is enforced by class presence, not media query or build flag.
6. The editor god-component (`src/app/(dashboard)/dashboard/[serverId]/page.tsx`) must not grow in line count тАФ Appearance tab logic goes in `src/components/editor/appearance-tab.tsx`.

Source: CLAUDE.md architecture rules; CONTEXT.md D-13, D-14; REQUIREMENTS.md THEME-04
